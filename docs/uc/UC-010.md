# UC-010: Client Mouse Input Relay

- **UC ID:** UC-010
- **Title:** Client Mouse Input Relay
- **Priority:** P0
- **Estimated Complexity:** M

## Description

As a client user, I want my mouse movements, clicks, and scroll events captured on the
client and relayed to the host so that I can control the host's cursor and interact
with applications.

## Acceptance Criteria

1. Mouse movement, button clicks, and scroll events are captured and sent to the host
2. The host cursor moves in sync with client mouse movement
3. Input relay latency (mouse move to host-side cursor update) is under 2 ms on a LAN
4. All mouse buttons (left, right, middle, side buttons) are supported
5. Scroll wheel works in both vertical and horizontal directions
6. Mouse movement uses relative deltas (not absolute positioning) for gaming compatibility

## Technical Approach

Capture mouse events on the client, serialize them, send over the network to the host,
and inject them as synthetic input events. See [ADR-006](../adr/ADR-006.md) for the platform-specific
input capture mechanism.
