# UC-025: Host Wake-on-LAN

- **UC ID:** UC-025
- **Title:** Host Wake-on-LAN
- **Priority:** P1
- **Estimated Complexity:** M

## Description

As a client user, I want to wake up my host machine from the client app so that I can
start a streaming session without needing to physically interact with the host first.

## Acceptance Criteria

1. The client can send a Wake-on-LAN magic packet to a previously-paired host by name
2. After sending the wake packet, the client polls for the host to come online and
   reports progress to the user
3. If the host comes online within a configurable timeout (default: 60 seconds),
   the client connects automatically
4. If the timeout expires, the client reports the failure clearly with guidance
5. The host's MAC address is stored alongside its pairing record so no manual entry is needed
6. Wake-on-LAN can be triggered from the CLI: `rayview wake <host-name>`

## Technical Approach

Wake-on-LAN works by broadcasting a magic packet (6 × 0xFF followed by the target MAC
address repeated 16 times) as a UDP broadcast on port 9 (or 7). The client must know
the host's MAC address — this is stored in the pairing record during the initial
connection ([UC-016](UC-016.md)). After sending the packet, the client retries discovery ([UC-014](UC-014.md))
until the host appears or the timeout elapses.
