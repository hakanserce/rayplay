# UC-002: Host Video Encoding

- **UC ID:** UC-002
- **Title:** Host Video Encoding
- **Priority:** P0
- **Estimated Complexity:** L

## Description

As a host user, I want captured screen frames to be hardware-encoded into a compressed
video stream so that they can be efficiently transmitted to the client with minimal latency.

## Acceptance Criteria

1. Frames are encoded using HEVC (H.265) via NVENC on Nvidia GPUs (RTX 2060 and later)
2. Encoding latency is under 5 ms per frame at 1080p60 and 4K60
3. The encoded HEVC stream is compatible with VideoToolbox on macOS for hardware decoding
4. Bitrate is configurable (default: auto based on resolution and frame rate)
5. Encoding does not cause visible frame drops under normal gaming load
6. The encoded output is chunked into network-friendly packet sizes

## Technical Approach

Use NVENC to encode HEVC directly from the DXGI-captured frame with minimal copying.
HEVC is the default and first-supported codec — it offers better quality-per-bit than
H.264 and is hardware-accelerated on all Nvidia GPUs from the RTX 2060 generation onward.
See ADR-001 for the zero-copy architecture and ADR-004 for the codec FFI approach.
Fallback to other codecs (if HEVC NVENC is unavailable) is handled by UC-023.

## Dependencies

- UC-001 (Host Screen Capture)
- **ADR-001** (zero-copy graphics architecture)
- **ADR-004** (video codec FFI approach)
