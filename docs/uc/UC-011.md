# UC-011: Exclusive Input Mode

- **UC ID:** UC-011
- **Title:** Exclusive Input Mode
- **Priority:** P0
- **Estimated Complexity:** M

## Description

As a client user, I want to toggle an exclusive input mode that traps my mouse cursor
inside the client window and forwards all input (including system keys) to the host
so that the streaming experience feels like sitting in front of the host machine.

## Acceptance Criteria

1. A configurable hotkey toggles exclusive input mode on and off
2. When active, the mouse cursor is confined to the client window and hidden
3. When active, all keyboard input (including system keys like Alt+Tab) is forwarded
   to the host instead of being handled by the client OS
4. When active, the client window captures mouse input even at screen edges
5. Exiting exclusive mode restores normal client OS input behavior immediately
6. A clear visual indicator shows when exclusive mode is active

## Technical Approach

This builds on UC-009 and UC-010 by adding cursor confinement and system key
interception on the client. See ADR-006 for the platform-specific mechanisms
needed to trap input at the OS level.

## Dependencies

- UC-009 (Client Keyboard Input Relay)
- UC-010 (Client Mouse Input Relay)
- **ADR-006** (input capture mechanism)
