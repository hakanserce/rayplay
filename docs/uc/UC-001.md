# UC-001: Host Screen Capture

- **UC ID:** UC-001
- **Title:** Host Screen Capture
- **Priority:** P0
- **Estimated Complexity:** M

## Description

As a host user, I want RayHost to capture my primary display's screen content in
real time so that it can be fed into the streaming pipeline.

## Acceptance Criteria

1. RayHost captures the host's primary display at the native resolution
2. Capture runs continuously at a configurable frame rate (default 60 fps)
3. Captured frames are delivered to the encoding stage with less than 1 ms capture latency
4. Capture does not visibly degrade the host's desktop or gaming performance
5. Capture works while full-screen games are running

## Technical Approach

Implement platform-specific screen capture for Windows. The capture mechanism should
provide raw frame data suitable for hardware encoding. See [ADR-001](../adr/ADR-001.md) for the zero-copy
architecture decision.

## Dependencies

- None (first UC in the pipeline)
- **[ADR-001](../adr/ADR-001.md)** (zero-copy graphics architecture) should be resolved first

## Notes

- Only the primary monitor is captured in this UC; multi-monitor is [UC-019](UC-019.md)
- The capture format and color space should match what the encoder expects to avoid
  unnecessary conversions
