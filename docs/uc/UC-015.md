# UC-015: Session Management

- **UC ID:** UC-015
- **Title:** Session Management
- **Priority:** P1
- **Estimated Complexity:** M

## Description

As a user, I want the host and client to negotiate stream parameters and manage the
session lifecycle (connect, disconnect, reconnect) so that streaming sessions are
reliable and configurable.

## Acceptance Criteria

1. Client and host perform a handshake that negotiates resolution, frame rate, and codec
2. The negotiated parameters are the best match of client and host capabilities
3. Either side can cleanly disconnect, and the other side handles it gracefully
4. The client can reconnect to the same host without restarting the host
5. Session state (active, disconnected, reconnecting) is visible to the user
6. Connection timeout is configurable with a sensible default

## Technical Approach

Implement a session control channel alongside the media transport. The handshake
exchanges capability information and agrees on stream parameters before media flows.
See ADR-003 for the transport protocol decision — the session control channel maps
naturally to a reliable stream in the chosen protocol.
