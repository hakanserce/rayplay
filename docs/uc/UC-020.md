# UC-020: Gamepad Input Support

- **UC ID:** UC-020
- **Title:** Gamepad Input Support
- **Priority:** P2
- **Estimated Complexity:** M

## Description

As a client user, I want to use a gamepad (controller) connected to my client machine
and have its input forwarded to the host so that I can play games that require a controller.

## Acceptance Criteria

1. Gamepads connected to the client are detected automatically
2. All standard gamepad inputs are supported: buttons, triggers, thumbsticks, d-pad
3. Gamepad input is relayed to the host and injected as a virtual controller
4. Input latency for gamepad events is under 2 ms on a LAN
5. Multiple gamepads can be connected simultaneously
6. Analog inputs (triggers, sticks) maintain full precision

## Technical Approach

Extend the input capture and relay system from [UC-009](UC-009.md)/[UC-010](UC-010.md) to support gamepad
devices. The host creates virtual gamepad devices that receive the relayed input.

## Dependencies

- [UC-009](UC-009.md) (Client Keyboard Input Relay)
- [UC-010](UC-010.md) (Client Mouse Input Relay)
