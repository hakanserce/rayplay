# UC-005: Client Frame Rendering

- **UC ID:** UC-005
- **Title:** Client Frame Rendering
- **Priority:** P0
- **Estimated Complexity:** M

## Description

As a client user, I want decoded video frames rendered to a window on my macOS screen
so that I can see the host's display in real time.

## Acceptance Criteria

1. Decoded frames are rendered to a resizable application window
2. Rendering latency from decoded frame to screen presentation is under 2 ms
3. The window displays stream content scaled to fill the viewport (stretch-to-fit)
4. Frame rendering is smooth with no visible tearing or stuttering at 60 fps
5. The window can be toggled to full-screen mode

## Technical Approach

Use a GPU-accelerated rendering pipeline to display decoded frames in a native
macOS window. See ADR-005 for the rendering framework decision.

## Dependencies

- UC-004 (Client Video Decoding)
- **ADR-005** (window/rendering framework)
