# UC-021: Client-Side Cursor Rendering

- **UC ID:** UC-021
- **Title:** Client-Side Cursor Rendering
- **Priority:** P2
- **Estimated Complexity:** M

## Description

As a client user, I want the mouse cursor rendered on the client side with minimal
latency so that cursor movement feels instant even when video frames are slightly delayed.

## Acceptance Criteria

1. The host sends cursor shape and position updates separately from the video stream
2. The client renders the cursor locally based on mouse input, not video frames
3. Cursor rendering latency is under 1 ms (feels instant)
4. Cursor shape changes (arrow, hand, resize, etc.) are reflected on the client
5. The client cursor position is reconciled with the host cursor when frames arrive

## Technical Approach

Decouple cursor rendering from the video pipeline. The host sends cursor shape updates
as lightweight messages. The client renders the cursor locally using mouse deltas,
overlaid on the video frame.
