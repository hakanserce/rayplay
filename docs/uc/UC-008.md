# UC-008: End-to-End Video Streaming

- **UC ID:** UC-008
- **Title:** End-to-End Video Streaming
- **Priority:** P0
- **Estimated Complexity:** M

## Description

As a user, I want to start RayHost on my Windows machine and RayView on my macOS
machine and see the host's screen rendered on the client at 60 fps so that the
complete video streaming pipeline works end to end.

## Acceptance Criteria

1. Starting RayHost and connecting RayView produces a live video stream on the client
2. The stream runs at 60 fps with no visible stuttering under normal LAN conditions
3. Total glass-to-glass latency (capture to display) is under 16 ms on a local network
4. The stream recovers from brief network interruptions without crashing
5. Both host and client can be stopped and restarted without requiring the other to restart
6. Stream quality is visually acceptable for gaming (no persistent artifacts)

## Technical Approach

This is an integration UC that validates the full pipeline: UC-001 through UC-007
working together. Focus is on measuring end-to-end latency and ensuring all stages
connect correctly.

## Dependencies

- UC-006 (Host CLI)
- UC-007 (Client CLI)
