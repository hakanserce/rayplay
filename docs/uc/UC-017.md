# UC-017: Latency Metrics & Overlay

- **UC ID:** UC-017
- **Title:** Latency Metrics & Overlay
- **Priority:** P1
- **Estimated Complexity:** M

## Description

As a user, I want to see real-time latency metrics (capture, encode, network, decode,
render) displayed as an overlay on the client so that I can monitor streaming performance.

## Acceptance Criteria

1. The client measures and displays per-stage latency: capture, encode, network, decode, render
2. Total glass-to-glass latency is calculated and displayed
3. Metrics are shown as a toggleable on-screen overlay
4. Metrics update at least once per second
5. The overlay includes frame rate (actual vs target) and packet loss percentage
6. Metrics can be logged to a file for post-session analysis

## Technical Approach

Instrument each pipeline stage with timestamps. The host embeds capture and encode
timestamps in frame metadata; the client measures decode and render times locally.
Network latency is derived from round-trip measurements.
