# UC-004: Client Video Decoding

- **UC ID:** UC-004
- **Title:** Client Video Decoding
- **Priority:** P0
- **Estimated Complexity:** L

## Description

As a client user, I want received video data to be decoded on my macOS machine using
hardware acceleration so that frames are ready for display with minimal CPU usage and latency.

## Acceptance Criteria

1. Incoming video packets are decoded using the client's hardware decoder
2. Decoding latency is under 3 ms per frame
3. Decoded frames are delivered to the rendering stage in a GPU-friendly format
4. The decoder recovers gracefully from corrupted or missing packets without crashing
5. CPU usage remains low (under 10%) during active decoding

## Technical Approach

Use macOS hardware decoding capabilities to decode the video stream received via
UC-003. See ADR-004 for the codec FFI approach and ADR-005 for how decoded frames
integrate with the rendering framework.

## Dependencies

- UC-003 (Video Stream Transport)
- **ADR-004** (video codec FFI approach)
- **ADR-005** (window/rendering framework)
