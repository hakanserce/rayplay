# UC-012: Host Audio Streaming

- **UC ID:** UC-012
- **Title:** Host Audio Streaming
- **Priority:** P1
- **Estimated Complexity:** M

## Description

As a host user, I want the server to capture my system's audio output and stream it
to the client so that the client user can hear game sounds and other audio.

## Acceptance Criteria

1. The host captures system audio output (all sounds playing on the host)
2. Audio is encoded and transmitted to the client with low latency
3. Audio-to-network latency is under 5 ms
4. Capture works regardless of the audio source (games, media players, system sounds)
5. Audio capture does not mute or interfere with local host audio playback

## Technical Approach

Capture the host's audio loopback output, encode with a low-latency codec, and
transmit to the client. See [ADR-002](../adr/ADR-002.md) for the audio pipeline architecture and
[ADR-003](../adr/ADR-003.md) for the transport protocol.
