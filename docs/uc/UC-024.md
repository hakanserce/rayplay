# UC-024: Configuration Persistence

- **UC ID:** UC-024
- **Title:** Configuration Persistence
- **Priority:** P2
- **Estimated Complexity:** S

## Description

As a user, I want my streaming preferences saved to a configuration file so that
I do not need to reconfigure settings each time I start a session.

## Acceptance Criteria

1. Stream settings (resolution, frame rate, bitrate, codec, monitor) are saved to a config file
2. The config file is human-readable and editable
3. CLI arguments override saved settings when provided
4. Default settings are used when no config file exists
5. The config file location follows platform conventions (e.g., ~/.config/rayplay/)

## Technical Approach

Implement a configuration layer that loads from a TOML file, merges with CLI arguments,
and exposes the resolved settings to the application.

## Dependencies

- UC-006 (Host CLI)
- UC-007 (Client CLI)
