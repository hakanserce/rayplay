# UC-009: Client Keyboard Input Relay

- **UC ID:** UC-009
- **Title:** Client Keyboard Input Relay
- **Priority:** P0
- **Estimated Complexity:** M

## Description

As a client user, I want my keyboard input captured on the client and relayed to the
host so that keystrokes I type are executed on the host machine as if I were sitting
in front of it.

## Acceptance Criteria

1. Key presses and releases on the client are captured and sent to the host
2. The host receives and injects keyboard events so that applications respond normally
3. Input relay latency (key press to host-side injection) is under 2 ms on a LAN
4. All standard keys work including modifiers (Shift, Ctrl, Alt, Cmd/Win)
5. Key repeat behavior matches what the host would produce from a local keyboard
6. A configurable hotkey allows the user to release keyboard capture back to the client OS

## Technical Approach

Capture keyboard events on the client, serialize them, send over the network to the
host, and inject them as synthetic input events. See [ADR-006](../adr/ADR-006.md) for the platform-specific
input capture mechanism and [ADR-003](../adr/ADR-003.md) for the transport protocol.
