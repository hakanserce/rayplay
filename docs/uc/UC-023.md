# UC-023: Additional Codec Support

- **UC ID:** UC-023
- **Title:** Additional Codec Support
- **Priority:** P2
- **Estimated Complexity:** M

## Description

As a user, I want the option to use newer video codecs (such as HEVC or AV1) for
better quality at lower bitrates on hardware that supports them.

## Acceptance Criteria

1. The system supports at least one additional codec beyond the baseline H.264
2. Codec selection is negotiated during session handshake based on hardware capabilities
3. The user can override automatic codec selection via CLI option
4. Stream quality at equivalent bitrates is measurably better with newer codecs
5. Fallback to the baseline codec occurs automatically if the selected codec is unsupported

## Technical Approach

Extend the encoding (UC-002) and decoding (UC-004) stages to support additional
codecs. Session negotiation (UC-015) selects the best mutually supported codec.

## Dependencies

- UC-002 (Host Video Encoding)
- UC-015 (Session Management)
