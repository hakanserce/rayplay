# UC-026: Platform-Neutral Fallback Pipeline

- **UC ID:** UC-026
- **Title:** Platform-Neutral Fallback Pipeline
- **Priority:** P1
- **Estimated Complexity:** L

## Description

As a developer or user on a non-optimized platform, I want RayPlay to provide a
platform-neutral fallback for screen capture, video encoding, video decoding, and
frame rendering so that I can test and use RayPlay on platforms beyond the optimized
Windows host and macOS client — accepting higher latency in exchange for broader
compatibility.

## Acceptance Criteria

1. RayHost can capture, encode, and stream on any desktop platform (Linux, macOS)
   using a software-based pipeline when the optimized Windows/NVENC path is unavailable
2. RayView can decode and render a stream on any desktop platform (Linux, Windows)
   using a software-based pipeline when the optimized macOS path is unavailable
3. The fallback pipeline is selected automatically when the optimized path is not
   available on the current platform
4. The user can force the fallback pipeline via a CLI flag (e.g., `--software`)
   for testing purposes
5. End-to-end streaming works between any two desktop platforms using the fallback
   pipeline (e.g., Linux host → Linux client)
6. The fallback pipeline produces a visually correct stream at ≥30 fps at 1080p
   on a modern quad-core CPU
7. All existing integration tests pass when run against the fallback pipeline
