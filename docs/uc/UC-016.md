# UC-016: Connection Pairing & Security

- **UC ID:** UC-016
- **Title:** Connection Pairing & Security
- **Priority:** P1
- **Estimated Complexity:** M

## Description

As a host user, I want a PIN-based pairing flow so that only authorized clients can
connect to my host, preventing unauthorized access on shared networks.

## Acceptance Criteria

1. First-time connections require a PIN displayed on the host to be entered on the client
2. Once paired, the client is remembered and can reconnect without re-entering the PIN
3. The host can view and revoke paired clients
4. All stream data between paired devices is encrypted
5. Unpaired clients cannot view or interact with the host's stream
6. The pairing flow completes in under 10 seconds

## Technical Approach

Implement a pairing protocol where the host generates a short PIN, the client submits
it, and upon match, both sides establish an encrypted channel. Paired client identities
are persisted on the host. See [ADR-007](../adr/ADR-007.md) for the cryptographic approach (encryption
algorithm, key exchange mechanism, and identity storage format).
